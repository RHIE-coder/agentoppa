# classify.md — 주입할 자리 정하기: 어디에·얼마나 세게

일반화한 원칙을 *어디에 넣고 얼마나 세게* 적용할지 정한다. 두 가지 — ① **범위**(누구한테까지), ② **수단**(무엇으로). 실제로 넣는 법은 [route.md](route.md).

## ① 범위 — 배포 vs 내부

- **배포** — 이 교훈이 막는 게 *우리가 만들어내는 결과물*에도 나타난다 → 플러그인 쪽에 넣으면 **유저한테도 적용**된다.
- **내부** — *이 repo 살림*(README 표 모양 같은). 유저 결과물엔 안 나타남 → git 기록으로 충분.

> 판별: "이 교훈이 막는 게 *우리가 만들어내는 결과물*에 나타나나?" 그렇다=배포. repo 청소뿐=내부. 하나가 둘 다일 수 있다 — 나눠 처리.

**범위가 자리를 정한다:**

| 범위 | 어디에 | 규칙 | 검사기 | 훅 |
|---|---|---|---|---|
| **배포** (유저에도 + 우리 dev엔 `--plugin-dir`로) | `plugins/agentoppa/` | `always-on.md` | `skills/*/scripts/validate.mjs` | `hooks/` |
| **내부** (이 repo만, ship 안 됨 = 유저에게 배포 안 됨) | repo 루트 | `AGENTS.md` | `test/` | (git 훅 등) |

(범위 무관 — **로그**(이번 사건·추론)는 항상 `.agentoppa/hardening-log.md`.)

## ② 수단 — 강하고 싼 것부터

| 수단 | 무엇 / 효과 | 평소 컨텍스트 |
|---|---|---|
| **검사기** (+빨강/초록 테스트 = 나쁜 예는 실패(빨강)·정상은 통과(초록), 둘 다 확인) | 코드가 자동 검사해 어기면 실패시킴 (제일 셈) | **0** (돌기만) |
| **훅** | 특정 행동(저장·커밋 등) 순간 자동 실행 | **0** |
| **리뷰어** 한 줄 | 리뷰 담당 에이전트 지시문에 한 줄 | 리뷰 안에서만 |
| **Gate** 한 줄 | 어떤 단계를 '끝'이라 하기 전 통과 조건 | 그 스킬 쓸 때 |
| **문서** 한 줄 | 그 작업 설명서(`references/`)에 한 줄 (읽을 때만) | 그 작업 때만 |
| **항상 규칙** | 모든 세션에 늘 뜸 — 배포=`always-on.md` / 내부=`AGENTS.md` (한 줌만) | 항상 |

**위로 갈수록 강하면서 동시에 싸다.** 아래로 갈수록 약하고 비싸다 → 될 수 있으면 위로.

## 수단 결정 — 위에서부터 순차로 (이게 판정 프로세스)

강한 것부터 차례로 "이 교훈을 **이 수단으로 잡을 수 있나?**"를 묻고, *잡히는 가장 위*에 배정한다. 한 교훈을 여러 수단으로 쪼개도 된다.

1. **검사기** — *기계로 판정 가능한가?*(경로·패턴 등) → 가능하면 여기 + 빨강/초록 테스트.
2. **훅** — *특정 행동 순간에 자동으로 막을 일인가?*
3. **리뷰어** — *리뷰 단계에서 판단해야 하나?*
4. **Gate** — *특정 단계의 완료 조건으로 붙일 수 있나?*
5. **문서** — *그 작업을 읽을 때만 필요한 앎인가?*
6. **항상 규칙** — *어디에도 안 맞고 늘 살아야 하나?* (맨 아래 = 최후, 한 줌만)

원칙: 기계로 잡히면 1·2. 판단이면 3·4·5. 둘 다면 **쪼갠다**(기계 조각=검사기, 판단 조각=규칙/문서). *잡히는데 문서로 미루면* 아무도 안 읽어 헛일 / *억지로 기계화하면* 멀쩡한 것도 막는 오작동.

## 왜 위(자동 검사)부터냐

**맹점**(내가 *틀린 줄도 모르는* 실수)은 검색으로 못 막는다 — 찾아볼 생각 자체를 못 하니까. 검사기·훅은 *내가 안 찾아도* 그 순간 알아서 튀어나온다 → 맹점에 유일하게 듣는다. 반대로 문서·`AGENTS.md`는 *내가 읽을 때만* 도움(아는 실수엔 좋지만 맹점엔 약함). 게다가 자동 검사는 평소 컨텍스트를 0만큼 먹는다 → **"맹점 막기"와 "토큰 아끼기"가 같은 방향**.

## 한계 (솔직히)

- 취향·판단 교훈("이 구조 과하다")은 기계로 못 잡음 → 리뷰어/문서. 이 묶음은 작게 유지.
- 룰+테스트는 한 줄 메모보다 품이 든다 = *그게 장점*(제대로 일반화·검증하게 강제). **또 나거나 비싼** 종류만 검사기로. 한 번뿐인 건 그냥 고친 커밋으로.
