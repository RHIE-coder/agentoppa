    ├── interface.json         # slot spec
    └── always-on.md
.harness/ (Project Layer · this project)
.harness/
├── config.yaml        # config (required). setup lays it down, you fill slots
├── project/           # your impl (fill as you like)
│   ├── impl/          # impl modules
│   └── agents/        # helper agents
├── core/validate.mjs  # validator (the build adds it)
└── artifacts/         # outputs (appear when you run)

So another project just installs and the Core builds .harness itself (without AgentOppa).

What it's made of

The tools inside

AgentOppa comes in two groups: tools that drive the flow, and tools that make the parts of a harness.

Tools that drive the flow

agent-engineergenerator
Runs the whole build — interview, assemble, package, verify. Calls intent-interview and the ccc-* parts inside.
intent-interviewinterview
Interviews you one question at a time to pin down intent. Auto-included as agent-engineer's first step; also usable standalone when you just want to clarify intent.
self-hardenhardening
Freezes a corrected mistake into a permanent guard (validator·hook·rule), so the same mistake can't recur.

ccc = create-claude-codex — makes the parts below work identically on both Claude & Codex.

Tools that make the parts (ccc-*)

ccc-skillsskill
A skill is a how-to for a task. It's auto-invoked on a matching request and runs the steps. Makes skills that work the same on both tools.
ccc-memorymemory
Memory is the rules an agent follows in every session (e.g. AGENTS.md). Keeps both tools reading the same rules.
ccc-agentssubagent
A subagent is a dedicated agent for a role (e.g. code review). The main agent delegates that work. Built for both tools.
ccc-hookshook
A hook runs automatically at set moments (after an edit, before stop). Makes hooks that work on both tools.
ccc-pluginplugin
A plugin bundles skills·hooks·subagents into one installable unit, shipped to both Claude & Codex marketplaces.
