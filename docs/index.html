Q. 프로젝트마다 달라질 부분이 있나요?
> 테스트 도구요 (프로젝트마다 달라요)
좋아요. 그 자리는 빈자리로 두고 조립할게요.
✓ 단계별 스킬 조립 (ccc-skills)
✓ Claude·Codex 양쪽으로 포장 (ccc-plugin)
✓ .agentoppa/ 재사용 Core로 빌드 + 검증 통과
완성. 다른 프로젝트도 이 Core를 가리켜 쓸 수 있어요.
참고 · 이렇게 설명만 하면 스킬이 자동으로 켜집니다 (/agent-engineer 로 직접 불러도 됨). 면담(intent-interview)은 agent-engineer 안에서 자동으로 일어나요. 따로 부를 필요 없습니다.

2Core 안엔 무엇이, 설치하면 어떻게 되나

.agentoppa/ (Core Layer · agent-engineer가 만든 재사용 틀)
.agentoppa/
├── .claude-plugin/ .agents/   # 두 도구 마켓
└── plugins/my-harness-workflow/
    ├── skills/{spec,tdd,review}/  # 워크플로우 단계
    ├── skills/setup/          # .harness/<하네스> 를 자동으로 만듦
    ├── interface.json         # 빈자리 명세
    └── always-on.md
.harness/<하네스>/ (Project Layer · 이 프로젝트 층)
.harness/my-harness-workflow/
├── config.yaml        # 설정 (필수). setup이 깖, 빈자리만 채움
├── project/           # 내 구현 (유저가 입맛대로 채움)
│   ├── impl/          # 구현 모듈
│   └── agents/        # 보조 에이전트
├── core/validate.mjs  # 검사기 (빌드가 넣음)
└── artifacts/         # 결과물 (돌리면 나옴)

그래서 다른 프로젝트는 설치만 하면 Core가 스스로 .harness/<하네스>를 만들어 줍니다 (AgentOppa 없이).

프로젝트 B · Claude Code (AgentOppa 없음)
> /plugin install my-harness-workflow
✓ 설치됨: 단계 스킬(spec·tdd·review) + setup
> 이 하네스 붙여줘
🔧 setup: .harness/my-harness-workflow 를 만듭니다 (이 플러그인만으로)
✓ 만듦: .harness/my-harness-workflow/config.yaml (core: my-harness-workflow)
채울 빈자리: test-runner
프로젝트를 보니 playwright 가 있네요. 그걸로 채울게요.
✓ test-runner: "npx playwright test"
완료. 이제 /spec 부터 쓰면 됩니다. (AgentOppa 한 번도 안 씀)

같은 Core 하나를 여러 프로젝트가 씁니다. 프로젝트마다 다른 건 빈자리(bindings)뿐입니다. Core의 setup이 .harness/<하네스>를 만들고, 프로젝트를 살펴 채웁니다. Core를 한 번 고치면 가리키는 모든 프로젝트에 반영됩니다.

✓

만들 땐 AgentOppa(Maker), 쓸 땐 필요 없습니다. AgentOppa는 Core를 만들어 주는 컴파일러예요. 다 만든 Core(.agentoppa)는 혼자 도는 플러그인이라, 다른 프로젝트는 그걸 설치만 하면 그 안 setup이 .harness/<하네스>를 알아서 만들어 줍니다.

안에 든 도구들

AgentOppa는 두 묶음으로 되어 있습니다. 전체 흐름을 진행하는 도구, 그리고 하네스를 이루는 부품을 만드는 도구.

흐름을 진행하는 도구

agent-engineer생성기
면담부터 조립·포장·검증까지, 하네스를 만드는 전 과정을 진행합니다. 아래 intent-interview와 ccc-* 부품을 안에서 불러 씁니다.
intent-interview면담
무엇을 만들지 한 번에 한 질문씩 캐물어 의도를 또렷이 정리합니다. agent-engineer의 첫 단계로 자동 포함되며, 의도만 정리하고 싶을 땐 단독으로도 씁니다.
self-harden단단해지기
한 번 바로잡은 실수를 다시는 못 하게 영구 가드(자동 검사·훅·규칙)로 굳혀, 같은 실수의 재발을 막습니다.

ccc = create-claude-codex 아래 부품들을 Claude·Codex 양쪽에서 똑같이 동작하도록 만든다는 뜻입니다.

부품을 만드는 도구 (ccc-*)

ccc-skills스킬
스킬은 특정 작업의 절차를 담은 설명서입니다. 관련된 요청이 오면 자동으로 불려와, 정해진 방식대로 작업을 수행합니다. 양쪽 도구에서 동일하게 도는 스킬을 만듭니다.
ccc-memory메모리
메모리는 에이전트가 모든 세션에서 늘 따르는 규칙·지침입니다(예: AGENTS.md). 양쪽 도구가 같은 규칙을 읽도록 정리합니다.
ccc-agents서브에이전트
서브에이전트는 특정 역할을 전담하는 별도 에이전트입니다(예: 코드 리뷰 전담). 본체가 그 역할의 작업을 위임합니다. 양쪽 도구용으로 만듭니다.
ccc-hooks훅
훅은 정해진 시점(예: 파일 편집 직후, 작업 종료 전)에 자동으로 실행되는 명령입니다. 양쪽 도구에서 동작하는 훅을 만듭니다.
ccc-plugin플러그인
플러그인은 스킬·훅·서브에이전트를 하나로 묶어 설치·배포하는 단위입니다. Claude·Codex 양쪽 마켓에 함께 싣습니다.
